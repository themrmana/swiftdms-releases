# Part B — NVIDIA Profile Inspector

## Why these three settings aren't scripted

NVIDIA Profile Inspector reads/writes the driver's profile database via the
NVAPI library. The setting *IDs* (e.g. `0x10835000`) and the *value encodings*
(some are DWORDs, some are bit-packed enums, some shift between drivers) are
not part of a stable, documented public API. A `.nip` import generated by
guessing IDs can:

- Apply to the wrong setting silently
- Leave a setting in a state the NVIDIA Control Panel can't represent
- Mismatch on the next driver release and ghost across reinstalls

The 30 seconds of clicking saves itself many times over in not chasing weird
profile state later. So Part B is: the script downloaded the tool, took a
backup, and opened the GUI; you make three clicks below.

## Pre-flight

`scripts/04-prep-nvidia-profile-inspector.ps1` should already have:

- Downloaded the tool to `gaming-pc-tuning/tools/nvidiaProfileInspector/`
- Backed up the current global profile to `nvpi-backup-<timestamp>.nip` in that
  same folder
- Opened the GUI

If not, run it now (from the `scripts/` folder, elevated PowerShell):

```powershell
.\04-prep-nvidia-profile-inspector.ps1
```

## In the GUI

The dropdown at the top should show **_GLOBAL_DRIVER_PROFILE (Base Profile)**.
Leave it there — these changes are global, not per-game.

### 1. Max Frame Rate → 116

- Find the section **"5 — Common"**.
- Setting: **Frame Rate Limiter V3** (newer builds may just say
  **"Frame Rate Limit"** under "Common").
- Change from `Off (0)` to **`116`**.
- Why 116 and not 120: the LG NANO86's G-Sync Compatible window is roughly
  48–120 Hz. Capping a few frames below the panel max keeps us inside the VRR
  range continuously, so V-Sync (set next) never has to engage to clamp the top
  end. The cap is what eliminates tearing; G-Sync handles everything below.

### 2. Vertical Sync → On

- Section: **"2 — Sync and Refresh"**.
- Setting: **Vertical Sync**.
- Change to **`On`** (some builds label this `Force on (0x2A23DD08)`).
- Why: with G-Sync + a sub-refresh frame cap, this is a tear backstop for the
  rare case the GPU briefly exceeds the cap. It does NOT add latency in that
  configuration — Reflex / Low Latency Mode keeps the queue empty.

### 3. Low Latency Mode → Ultra

- Section: **"5 — Common"**.
- Setting: **Low Latency Mode** (the global mirror of the Control Panel's
  per-game setting). Some builds label this **"NULL_Mode"** or
  **"Ultra Low Latency"**.
- Change to **`Ultra`** (or `Ultra Low Latency`).
- Why: keeps render queue depth at 1, drops average frame latency a couple of
  ms. Reflex (in-game) trumps this when present; this is the fallback for
  games without Reflex.

### Apply + save

- Click the **disk icon** in the toolbar (top-right area) — **Apply changes**.
- Then **File → Export user defined profiles…** and save next to the backup
  as `nvpi-after-<date>.nip`. This is your documentation of the new state.

## Things to deliberately NOT change globally

- DLSS / Frame Generation / Reflex — set per-game in each game's menu.
- Anisotropic filtering, texture quality, antialiasing — set per-game.
- Threaded optimization, shader cache — leave at driver defaults.

## How to revert

```powershell
cd gaming-pc-tuning\tools\nvidiaProfileInspector
.\nvidiaProfileInspector.exe -import .\nvpi-backup-<timestamp>.nip
```

Then re-open the GUI and **File → Restore user defaults** if you also want to
clear any setting changes since the backup.
